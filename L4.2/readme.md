1. Сборка проекта
Скомпилируйте исходный код в исполняемый файл:

go build -o dgrep main.go

2. Подготовка тестовых данных
Создайте тестовый файл test.txt с набором строк. Можете использовать эту команду для быстрой генерации:

[INFO] Запуск системы...
[DEBUG] Подключение установлено.
[INFO] Маскот Go — это Gopher!
[WARNING] Память загружена на 70%.
[ERROR] Ошибка конфигурации.
[INFO] Gopher роет тоннель.
[DEBUG] Пользователь авторизован.
[INFO] Еще одна скучная строка.
[WARNING] Gopher прячется здесь!
[DEBUG] Завершение работы.



3. Запуск серверов (Воркеров)
Откройте три новых окна терминала и запустите серверы на разных ports. Они будут висеть в ожидании задач:

Терминал 1:

./dgrep -mode=server -port=8081


Терминал 2:

./dgrep -mode=server -port=8082


Терминал 3:

./dgrep -mode=server -port=8083
4. Запуск поиска (Координатор)
Откройте четвертый терминал и запустите клиентскую часть, передав ей адреса серверов, искомое слово и файл:

./dgrep -mode=client -servers="localhost:8081,localhost:8082,localhost:8083" -pattern="Gopher" test.txt

Ожидаемый результат: Координатор сообщит об успешном достижении кворума и выведет найденные строки.

 5. Тестирование отказоустойчивости (Кворум)
Система работает по принципу кворума (N/2) + 1. Для 3 серверов нужно минимум 2 ответа.

Мягкий отказ (Кворум есть):
Перейдите в Терминал 1 (порт 8081) и нажмите Ctrl+C, чтобы убить сервер.
Снова запустите команду координатора.
Результат: Поиск завершится успешно, так как 2 из 3 серверов живы.
Критический отказ (Кворума нет):
Перейдите в Терминал 2 (порт 8082) и тоже нажмите Ctrl+C.
Снова запустите команду координатора.
Результат: Программа выдаст фатальную ошибку ❌ Кворум НЕ достигнут, так как остался жив только 1 сервер.
